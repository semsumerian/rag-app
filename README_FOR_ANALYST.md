# RAG Application - Описание системы для бизнес-аналитика

## Что такое эта система?

**RAG Application** — это интеллектуальный помощник для работы с документами. Представьте, что у вас есть сотни документов (инструкции, регламенты, отчёты), и вам нужно быстро найти в них ответ на вопрос. Вместо того чтобы читать всё вручную, вы загружаете документы в систему и задаёте вопросы обычным языком — ИИ сам найдёт нужную информацию и даст точный ответ с указанием источника.

### Простой пример
1. Вы загружаете 50 PDF-инструкций по работе с оборудованием
2. Спрашиваете: "Как настроить принтер HP LaserJet?"
3. Система ищет в документах, находит нужную инструкцию и отвечает пошагово
4. Показывает, из какого именно документа взята информация

## Как это работает? (Без технических деталей)

### Этап 1: Загрузка документов
- Вы перетаскиваете файлы (PDF, Word, текстовые файлы) в специальную область
- Система "читает" содержимое каждого файла
- Разбивает текст на небольшие кусочки ("чанки") по 1000 слов
- Каждый кусочек превращает в числовой код ("эмбеддинг") — это как "отпечаток пальца" текста

**Аналогия:** Представьте библиотекаря, который читает все книги и составляет карточки с кратким содержанием каждой главы.

### Этап 2: Поиск информации
Когда вы задаёте вопрос:
1. Ваш вопрос тоже превращается в числовой код
2. Система сравнивает его с кодами всех кусочков документов
3. Находит 5 самых похожих по смыслу кусочков
4. Отправляет их вместе с вопросом в ИИ

**Аналогия:** Библиотекарь смотрит на ваш вопрос и находит 5 карточек, которые больше всего похожи по смыслу.

### Этап 3: Генерация ответа
- ИИ получает вопрос + найденные кусочки документов
- Анализирует информацию
- Формулирует понятный ответ
- Указывает, из каких документов взята информация

**Аналогия:** Эксперт читает найденные главы и пишет вам краткий ответ с ссылками на источники.

## Состав системы

### 1. Серверная часть (Backend)
**Что делает:** Хранит документы, обрабатывает запросы, общается с ИИ

**Основные функции:**
- Принимает файлы от пользователей
- Извлекает текст из PDF и Word
- Разбивает текст на кусочки
- Создаёт "отпечатки" текста (эмбеддинги)
- Ищет релевантную информацию
- Отправляет запросы к ИИ-модели

**Технологии:** Node.js, Express, TypeScript

### 2. Клиентская часть (Frontend)
**Что делает:** Интерфейс, который видит пользователь

**Основные функции:**
- Форма загрузки файлов (drag & drop)
- Список загруженных документов
- Окно чата для вопросов
- Отображение ответов с форматированием
- Переключение темы (тёмная/светлая)

**Технологии:** React, TypeScript, Vite

### 3. ИИ-модель (LLM)
**Что делает:** Генерирует ответы на основе найденной информации

**Используется:** Qwen 3 VL 8B (или аналогичная модель)
- Работает локально через LM Studio
- Понимает русский язык
- Форматирует ответы (заголовки, списки, таблицы)

### 4. Векторное хранилище
**Что делает:** Хранит "отпечатки" текста для быстрого поиска

**Принцип:** Каждый кусочек текста = набор чисел. Похожие по смыслу тексты имеют близкие числа.

## Бизнес-процесс использования

### Сценарий 1: Быстрый поиск в документации
1. **Актор:** Сотрудник техподдержки
2. **Цель:** Найти решение проблемы клиента
3. **Действия:**
   - Загружает базу знаний (50 PDF)
   - Задаёт вопрос: "Ошибка 0x80070005 при установке"
   - Получает инструкцию с шагами из документации
   - Видит, из какого файла взята информация

### Сценарий 2: Анализ регламентов
1. **Актор:** Менеджер проекта
2. **Цель:** Проверить соответствие процесса регламенту
3. **Действия:**
   - Загружает корпоративные регламенты
   - Спрашивает: "Какие сроки согласования для проектов до 1 млн?"
   - Получает точный ответ с указанием пункта регламента

### Сценарий 3: Обучение новых сотрудников
1. **Актор:** Новичок в компании
2. **Цель:** Быстро разобраться в процессах
3. **Действия:**
   - Загружает инструкции и гайды
   - Задаёт вопросы на естественном языке
   - Получает структурированные ответы

## Пользовательский интерфейс

### Экран авторизации
- Логин/пароль: admin/admin
- Переключатель темы (тёмная/светлая)

### Главный экран (3 колонки на десктопе)

**Левая колонка:**
1. **Загрузка документов** — область drag & drop
2. **Загруженные документы** — список с возможностью удаления
3. **Информация о системе** — технические детали

**Правая колонка:**
- **Чат с документами** — окно диалога с ИИ
  - История сообщений
  - Поле ввода вопроса
  - Источники информации (показываются под ответом)

### Адаптивность
- **Десктоп:** Две колонки рядом
- **Планшет:** Колонки друг под другом
- **Мобильный:** Одна колонка, оптимизированный интерфейс

- **Снижение ошибок:** Точные ответы с указанием источников
- **Масштабируемость:** Работает с любыми объёмами документов

### Для пользователей
- **Простота:** Интерфейс как в обычном мессенджере
- **Естественность:** Вопросы задаёте обычным языком

- **Удобство:** Работает на любых устройствах

## Требования к инфраструктуре

### Минимальные
- Компьютер с Windows 10/11
- 8 ГБ RAM
- Node.js установлен
- LM Studio с загруженной моделью

### Рекомендуемые
- 16+ ГБ RAM
- SSD диск
- Видеокарта для ускорения ИИ (опционально)

## Безопасность и доступ

### Локальный доступ
- Приложение работает на вашем компьютере
- Доступ только с этого ПК
- Порты закрыты для внешнего мира

### Удалённый доступ (опционально)
- Через Cloudflare Tunnel
- Временный защищённый URL
- HTTPS шифрование
- URL меняется при каждом запуске

## Ограничения системы

1. **Форматы файлов:** Только PDF, DOCX, TXT
2. **Размер файлов:** Зависит от настроек сервера
3. **Языки:** Лучше всего работает с русским и английским
4. **Контекст:** ИИ видит только загруженные документы
5. **Хранение:** Данные хранятся локально, нет облачной синхронизации
